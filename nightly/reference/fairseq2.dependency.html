fairseq2.dependency

fairseq2 relies on the dependency inversion principle to maintain a clean, well-tested, and extensible code base.

This module contains the abstract classes DependencyResolver and DependencyContainer, as well as the StandardDependencyContainer class, as a lightweight dependency injection API.

class fairseq2.dependency.DependencyResolver[source]

extends ABC

resolve[T](kls: type[T], key: str | None = None) -> T

Return the singleton dependency of type T. If key is not None, the singleton dependency with the specified key will be returned.

If a dependency of type T, or a dependency of type T with key cannot be found, a LookupError is raised.

abstract resolve_optional(kls: type[T], key: str | None = None) → T | None[source]

Return the singleton dependency of type T similar to resolve(), but return None instead of raising a LookupError if the dependency is not found.

abstract resolve_all(kls: type[T]) → Iterable[T]:[source]

Return all singleton dependencies of type T that have no associated key.

If multiple singleton dependencies of type T are registered, resolve() only returns the last registered one. In contrast, resolve_all() returns them all, in the order that they were registered.

If no dependency can be found, an empty iterable will be returned.

abstract resolve_all_keyed(kls: type[T]) → Iterable[tuple[str, T]][source]

Return all singleton dependencies of type T that have an associated key.

This method behaves similar to resolve_all(), but returns an iterable of key-dependency pairs instead.

class fairseq2.dependency.DependencyContainer[source]

extends DependencyResolver

register[T](kls: type[T], factory: DependencyFactory[T], key: str | None = None) -> None

Register a singleton dependency of type T.

factory is expected to return an instance of type T, or None if the instance cannot be created. It should follow the protocol:

class DependencyFactory(Protocol[T]):
    def __call__(self, resolver: DependencyResolver) -> T | None:
        ...

resolver passed to factory can be used to resolve the dependencies needed to construct the instance itself.

Optionally key can be specified. In this case, the same key must be passed to resolve() to return the dependency.

If multiple singleton dependencies of type T, optionally with the same key, are registered, resolve() will return the last registered one.

register_instance[T](kls: type[T], obj: T, key: str | None = None) -> None

Register an existing singleton dependency of type T.

Other than registering obj instead of a factory, the method behaves the same as register().

final class fairseq2.dependency.StandardDependencyContainer[source]

implements DependencyContainer

This is the standard implementation of DependencyContainer and transitively of DependencyResolver.

from abc import ABC, abstractmethod

from fairseq2.dependency import DependencyResolver, StandardDependencyContainer

container = StandardDependencyContainer()

# The interface
class Foo(ABC):
    @abstractmethod
    def foo(self) -> None:
        ...

# The implementation
class FooImpl(Foo):
    def foo(self) -> None:
        pass

# The factory
def create_foo(resolver: DependencyResolver) -> Foo:
    assert resolver is container

    return FooImpl()

container.register(Foo, create_foo)

foo = container.resolve(Foo)

assert isinstance(foo, FooImpl)
